---
title: Reviewing Generated Styles
description: Generated styling still needs boundaries humans can inspect.
---

Generated code can make a first draft appear quickly.

The review still belongs to a human.

Someone still has to decide which parts are safe to customize, which selectors
are accidental, and whether a future DOM change will break a theme.

AI did not create that maintenance problem. It made the problem easier to
notice.

If AI creates a component library, humans still need to answer:

- which styling surfaces are public;
- which selectors are accidental;
- why a style won;
- whether a DOM refactor breaks themes;
- where a generated value came from.

Those questions are easier to review when the boundaries have names.

Slots are supported styling hooks. Scopes are grouped overrides. Components
decide where overrides attach. `combineStyle` merges the inputs before render.
Debugging traces generated CSS back to the TypeScript that produced it.

AI can draft.

Humans can trace.

The last page gives those review questions a small set of names.
