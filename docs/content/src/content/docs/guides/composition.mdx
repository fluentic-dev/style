---
title: Composition
description: Compose style chains, scopes, and css props intentionally.
---

Composition in Fluentic Style happens at the layer that owns the decision.

## Builder composition

```ts
const base = style({
  borderRadius: 6,
});

const card = style()
  .merge(base)
  .hover({
    transform: 'translateY(-1px)',
  });
```

Use builder composition for shared standalone style fragments.

## Scope composition

```ts
const theme = style.scope([
  baseTheme,
  compactTheme,
]);
```

Scopes can contain other scope items and slot overrides.

Resolve scopes with the component styles they target:

```ts
const css = combineStyle(buttonStyles, theme(buttonStyles.root));
```

## Render composition

```tsx
<button css={[css.root, isLoading && css.loading, props.css]} />
```

Use render composition for conditionals that depend on current props or state.

## Prefer the right layer

| Need | Use |
| --- | --- |
| A reusable component part | `style.slot` |
| A visual fragment | `style` |
| A component theme | `style.scope` |
| Applying a theme to component styles | `combineStyle(styles, ...targets)` |
| A one-off element override | `css` prop |
| A render-time conditional class | `css={[condition && item]}` |
