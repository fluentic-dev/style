---
title: Babel
description: Notes for Babel-based projects.
---

Fluentic's public integrations now use bundler adapters rather than a public
Babel plugin subpath.

If your project uses Babel through Webpack, use the [Webpack](webpack) adapter
and keep Babel responsible for React and TypeScript:

```js title="babel-loader options"
export default {
  presets: [
    ['@babel/preset-react', {
      runtime: 'automatic',
      importSource: '@fluentic/style/plugin/jsx',
    }],
    '@babel/preset-typescript',
  ],
};
```

For other Babel-based build systems, use [Runtime-Only Mode](runtime-only-mode)
or build a custom integration around the compiler internals until a first-class
adapter exists.
