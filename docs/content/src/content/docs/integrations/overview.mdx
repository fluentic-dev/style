The integration layer wires Fluentic into the build system you already use.

Your component code still uses the same APIs:

```txt
style -> slot -> scope -> combineStyle
```

The integration handles the pieces that differ by build system: JSX runtime
wiring, development CSS, sourcemaps, runtime entries, and static extraction.

| Environment | Start here |
| --- | --- |
| Shared plugin options | [Plugin Options](options) |
| Next.js App Router | [Next.js](nextjs) |
| Vite + React | [Vite](vite) |
| Webpack | [Webpack](webpack) |
| Rspack | [Rspack](rspack) |
| Farm | [Farm](farm) |
| Parcel 2 | [Parcel](parcel) |
| No compiler plugin | [Runtime-Only Mode](runtime-only-mode) |
| Custom build system | [Custom Compiler](custom-compiler) |

Production apps should use a bundler adapter when one exists. Runtime-only usage
works with the JSX runtime configured, but it keeps more work in the browser.
