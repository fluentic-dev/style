React 18 or newer is a peer dependency. Fluentic Style imports React for its JSX
runtime and hooks, but your app provides the React version. Bundler packages are
optional peer dependencies, so you only install the integration you actually
use.

## Pick a mode

There are two useful starting modes. They use the same component code.

| Mode | Best for | Setup |
| --- | --- | --- |
| Runtime-only | Prototyping, tests, component libraries, unsupported bundlers, SSR experiments | JSX import source and `combineStyle` |
| Static extraction | Applications and production bundles where a supported bundler is available | Runtime setup plus a bundler plugin and virtual CSS imports |

Runtime-only is not a throwaway mode. It is the fallback that keeps the same
style definitions usable when static evaluation is not available. Static
extraction is an optimization path for chains the compiler can evaluate.

## TypeScript

Use the custom JSX runtime so the `css` prop is accepted on DOM and SVG elements.

```json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "@fluentic/style/plugin/jsx"
  }
}
```

## Runtime-only check

After the JSX runtime is configured, this is enough to verify the runtime path:

```tsx
import { style, combineStyle } from '@fluentic/style';

const styles = {
  root: style.slot({
    display: 'inline-flex',
    borderRadius: 6,
    padding: '8px 12px',
  }),
};

export function Button() {
  const css = combineStyle(styles);
  return <button css={css.root}>Save</button>;
}
```

## Add extraction with Vite

Install the Vite React plugin if your app does not already have it:

<Tabs items={['pnpm', 'npm', 'Yarn', 'Bun']}>
  <Tab value="pnpm">

```bash
pnpm add -D @vitejs/plugin-react vite
```

  </Tab>
  <Tab value="npm">

```bash
npm install -D @vitejs/plugin-react vite
```

  </Tab>
  <Tab value="Yarn">

```bash
yarn add -D @vitejs/plugin-react vite
```

  </Tab>
  <Tab value="Bun">

```bash
bun add -d @vitejs/plugin-react vite
```

  </Tab>
</Tabs>

Then add the Fluentic Style plugin:

```ts
import { plugin as stylePlugin } from '@fluentic/style/plugin/vite';
import react from '@vitejs/plugin-react';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [
    stylePlugin(),
    react({
      jsxImportSource: '@fluentic/style/plugin/jsx',
    }),
  ],
});
```

Then import the virtual runtime module once near your application entry:

```tsx
import 'virtual:fluentic-style';
```

The runtime virtual module tells Fluentic Style whether the build is handled by
the plugin and whether CSS was extracted.

Other bundler adapters follow the same idea: configure the adapter, keep the JSX
runtime, and let the app bundler own final CSS output. See
[Integrations](../../integrations/overview/) for Vite, Rspack, Webpack, Farm,
and Next.js.
