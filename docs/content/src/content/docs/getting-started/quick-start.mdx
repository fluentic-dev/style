title: Quick Start
description: Start with `style()`, direct `css` prop composition, reusable style files, and scoped component styling.
---

Fluentic Style starts small. Write styles with `style()` and pass them directly
to the `css` prop. Use `combineStyle()` when a component needs scoped themes,
slot targets, or token overrides.

## 1. Write a style

```tsx
/** @jsxImportSource @fluentic/style */
import { style } from '@fluentic/style';

const button = style({
  display: 'inline-flex',
  alignItems: 'center',
  border: 0,
  borderRadius: 6,
  padding: '8px 12px',
  backgroundColor: '#2563eb',
  color: 'white',
  cursor: 'pointer',
}).hover({
  backgroundColor: '#1d4ed8',
});

export function Button() {
  return <button css={button}>Save</button>;
}
```

That is the core loop:

- `style(...)` creates a typed style builder.
- selector methods like `.hover(...)` keep state styles near the base style.
- `css={button}` puts the generated class on the element.

## 2. Combine styles

Keep related styles in one object and combine them with arrays.

```tsx
const styles = {
  button: style({
    display: 'inline-flex',
    alignItems: 'center',
    border: '1px solid rgb(15 23 42 / 0.14)',
    borderRadius: 6,
    padding: '8px 12px',
    backgroundColor: 'white',
    color: '#0f172a',
  }),
  primary: style({
    borderColor: '#2563eb',
    backgroundColor: '#2563eb',
    color: 'white',
  }),
  fullWidth: style({
    width: '100%',
  }),
};

export function Button(props: { primary?: boolean; fullWidth?: boolean }) {
  return (
    <button
      css={[
        styles.button,
        props.primary && styles.primary,
        props.fullWidth && styles.fullWidth,
      ]}
    >
      Save
    </button>
  );
}
```

The `css` prop accepts items, arrays, nested arrays, and falsy values. This makes
conditional styling feel like ordinary React composition.

## 3. Forward the `css` prop

Reusable components should let callers add element-level styles. Accept
`StyleProp`, then place it last in the `css` array so the caller can add layout or
one-off adjustments.

```tsx
/** @jsxImportSource @fluentic/style */
import { style } from '@fluentic/style';
import type { StyleProp } from '@fluentic/style';
import type { ReactNode } from 'react';

const styles = {
  button: style({
    display: 'inline-flex',
    alignItems: 'center',
    justifyContent: 'center',
    gap: 8,
    border: '1px solid rgb(15 23 42 / 0.14)',
    borderRadius: 6,
    padding: '8px 12px',
    backgroundColor: 'white',
    color: '#0f172a',
  }),
};

type ButtonProps = {
  children: ReactNode;
  css?: StyleProp;
};

export function Button(props: ButtonProps) {
  return <button css={[styles.button, props.css]}>{props.children}</button>;
}
```

Callers can pass another style builder directly:

```tsx
const pageStyles = {
  wideAction: style({
    minWidth: 160,
  }),
};

export function Page() {
  return <Button css={pageStyles.wideAction}>Publish</Button>;
}
```

## 4. Reuse styles across files

Style builders are normal values. Define common styles once, export them, and
compose them wherever you render.

```ts title="styles/common.ts"
import { style } from '@fluentic/style';

export const focusRing = style().focusVisible({
  outline: '2px solid #2563eb',
  outlineOffset: 2,
});

export const inlineControl = style({
  display: 'inline-flex',
  alignItems: 'center',
  justifyContent: 'center',
});
```

```ts title="Button.styles.ts"
import { style } from '@fluentic/style';
import { focusRing, inlineControl } from './styles/common';

export const buttonStyles = {
  root: style({
    border: '1px solid rgb(15 23 42 / 0.14)',
    borderRadius: 6,
    padding: '8px 12px',
    backgroundColor: 'white',
    color: '#0f172a',
  })
    .merge(inlineControl)
    .merge(focusRing),
  primary: style({
    borderColor: '#2563eb',
    backgroundColor: '#2563eb',
    color: 'white',
  }),
};
```

```tsx title="Button.tsx"
/** @jsxImportSource @fluentic/style */
import type { StyleProp } from '@fluentic/style';
import type { ReactNode } from 'react';
import { buttonStyles } from './Button.styles';

type ButtonProps = {
  children: ReactNode;
  css?: StyleProp;
  primary?: boolean;
};

export function Button(props: ButtonProps) {
  return (
    <button
      css={[buttonStyles.root, props.primary && buttonStyles.primary, props.css]}
    >
      {props.children}
    </button>
  );
}
```

This is enough for many app components: define styles, reuse style builders,
and compose them through `css`.

## 5. Add tokens when values become shared

When a value becomes part of your design language, make it a token.

```ts title="tokens.ts"
import { createToken } from '@fluentic/style';

export const color = {
  accent: createToken('#2563eb'),
  accentText: createToken('#ffffff'),
};
```

```ts title="Button.styles.ts"
import { style } from '@fluentic/style';
import { color } from './tokens';

export const buttonStyles = {
  root: style({
    backgroundColor: color.accent,
    color: color.accentText,
  }),
};
```

Tokens keep a stable identity, so later themes can retarget the same design
value without rewriting every component.

## Next steps

Use plain `style()` for styles that belong to one rendered element.

Use `style.slot(...)` when a component wants to publish a supported styling
target, such as `root`, `icon`, `label`, or `panel`.

Use `style.scope(...)` when you want to encapsulate a group of overrides for a
theme, variant, state, media query, or subtree.

Use `combineStyle(...)` when those scopes need to target component slots:

```tsx
const css = combineStyle(buttonStyles, primary(buttonStyles.root));
```

Continue with [Slots](../../core/slots/) and [Scopes](../../core/scopes/) when you are
ready to design a public component styling API.
