---
title: Recap
description: A short decision map for the Learn path.
---

Use the smallest Fluentic concept that matches the problem.

| Problem | Use |
| --- | --- |
| One element needs CSS | `style()` |
| A React element needs styles | `css` prop |
| One element needs optional styles | `css={[base, condition && next]}` |
| One element needs hover, focus, or media | `style().hover().focusVisible().media()` |
| Repeated local styles need a home | normal `*.styles.ts` exports |
| A reusable component has styleable parts | `style.slot()` |
| Outside code customizes component parts | `style.scope([...])` |
| A component resolves slots, variants, and themes | `combineStyle()` |
| Many scopes attach from the same root | `bindScope()` |
| A component publishes named theme hooks | `createToken()` / `createTokens()` |
| App code needs visible themeable literals | `createValues()` |
| A subtree needs different values | `createTheme()` |
| A component exports supported style targets | `exposeStyle()` |
| A rule needs to be traced back to source | source-aware styles, slots, scopes, and tokens |
| A production app needs CSS output | bundler integration and static extraction |

The main path is:

```txt
style -> css arrays -> chains
slots -> scopes -> combineStyle
component tokens -> app values -> createTheme
debug -> ship
```

Start simple.

Move to component APIs and themes only when the styling problem needs them.
