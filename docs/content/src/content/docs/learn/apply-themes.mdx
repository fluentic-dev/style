---
title: Apply Themes
description: Use createTheme() to override component tokens or app values for a subtree.
---

`createToken()`, `createTokens()`, and `createValues()` define themeable sources.

`createTheme()` provides different values for those sources.

For app values:

```ts
export const darkTheme = createTheme([
  color('#f5f7f6 | Canvas', '#101514'),
  color('#ffffff | Surface', '#18211f'),
  color('#15211d | Text', '#eef8f4'),
  color('#60716a | Muted', '#a6bbb3'),
  color('#d8e3df | Border', '#31443e'),
  color('#0f766e | Accent', '#5eead4'),
]);
```

For component tokens:

```ts
export const darkCardTheme = createTheme([
  cardTokens.color.surface('#111827'),
  cardTokens.color.text('#ffffff'),
  cardTokens.color.border('#374151'),
  cardTokens.color.accent('#60a5fa'),
]);
```

Apply a theme on the element that owns the subtree:

```tsx
const css = combineStyle(appStyles);

export function AppShell(props: { children: React.ReactNode }) {
  return <main css={[darkTheme, css.root]}>{props.children}</main>;
}
```

Everything inside the subtree reads the themed values.

The source styles do not change. They still refer to the same tokens or app
values.

Continue to [Theme App With Components](/docs/learn/theme-app-with-components/)
to put app themes and component slot themes together in real JSX.
