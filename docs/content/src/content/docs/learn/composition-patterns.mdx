---
title: Composition Patterns
description: Reuse scopes for variants, parent state, responsive changes, and incoming themes.
---

Now you have the main component pattern:

```txt
slots define targets
scopes provide overrides
combineStyle resolves them
```

The same pattern covers several common needs.

You may need:

- danger variant;
- selected state;
- parent hover;
- responsive density;
- nested component themes;
- conditional product tweaks.

Do not invent a separate styling API for each one.

Model each need as a scope:

```ts
const danger = style.scope([
  buttonStyles.root({ backgroundColor: '#dc2626', color: 'white' }),
]);

const parentHover = style.scope().hover([
  buttonStyles.root({ transform: 'translateY(-1px)' }),
]);

const compact = style.scope().media('(max-width: 700px)', [
  buttonStyles.root({ width: '100%' }),
  buttonStyles.label({ fontSize: 13 }),
]);
```

The slot calls live inside `style.scope([...])`.

That convention matters: the scope marks the value as a component override, even
when it only contains one slot override.

Resolve the scopes by attaching them from the root slot:

```tsx
const css = combineStyle(
  buttonStyles,
  props.danger && danger(buttonStyles.root),
  parentHover(buttonStyles.root),
  compact(buttonStyles.root),
);
```

An incoming `theme` prop is also slot-aware, so call it with the slot where the
theme should start:

```tsx
const css = combineStyle(
  buttonStyles,
  props.danger && danger(buttonStyles.root),
  parentHover(buttonStyles.root),
  compact(buttonStyles.root),
  props.theme?.(buttonStyles.root),
);
```

When several inputs all start from `buttonStyles.root`, `bindScope()` removes
the repeated calls:

```tsx
const css = combineStyle(
  buttonStyles,
  bindScope(
    buttonStyles.root,
    props.danger && danger,
    parentHover,
    compact,
    props.theme,
  ),
);
```

Use the types to make this clear in props:

```ts
type ButtonProps = {
  css?: StyleProp;
  theme?: StyleTheme;
};
```

`StyleProp` is for one rendered element.

`StyleTheme` is for slot-aware component customization.
