---
title: Styling Reusable Components
description: Name the component parts that callers may style.
---

Your button leaves the page and becomes a shared component:

```tsx
<Button icon={<SaveIcon />}>Save</Button>
```

Now other code wants to adjust it.

`css` or `className` can style the root.

They do not say much about the icon or label.

Selectors can target them. That can be fine locally, but now the caller knows
your DOM.

Extra props can work too, but every new styling need becomes a new prop.

The situation is really asking for a name:

> Which parts of this component should callers be allowed to style?

```ts title="button.styles.ts"
import { style } from '@fluentic/style';

export const buttonStyles = {
  root: style.slot({
    alignItems: 'center',
    border: 0,
    borderRadius: 8,
    display: 'inline-flex',
    gap: 8,
    padding: '8px 12px',
  }),
  icon: style.slot({
    flex: '0 0 auto',
  }),
  label: style.slot({
    fontWeight: 700,
  }),
};
```

That named public part is a slot.

Use `style.slot()` when you are willing to support that part as a styling API.

Slots do not expose DOM structure. They expose styling targets.

The next decision is which slots are actually public API and which styles stay
private.
