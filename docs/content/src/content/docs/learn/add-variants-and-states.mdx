---
title: Add Variants And States
description: Model variants, component states, and responsive component changes as scopes.
---

Once a component has slots, variants often affect more than one part.

Use scopes for those changes too:

```ts
const dangerButton = style.scope([
  buttonStyles.root({
    backgroundColor: '#dc2626',
    color: 'white',
  }),
  buttonStyles.icon({
    color: 'white',
  }),
]);

const compactButton = style.scope([
  buttonStyles.root({ padding: '4px 8px' }),
  buttonStyles.label({ fontSize: 13 }),
]);

const mobileButton = style.scope().media('(max-width: 700px)', [
  buttonStyles.root({ width: '100%' }),
  buttonStyles.label({ fontSize: 13 }),
]);
```

Resolve them with the component's other style inputs:

```tsx
const css = combineStyle(
  buttonStyles,
  bindScope(
    buttonStyles.root,
    props.danger && dangerButton,
    props.compact && compactButton,
    mobileButton,
    props.theme,
  ),
);
```

This keeps the model consistent:

```txt
variant -> scope
state   -> scope
theme   -> scope
resolve -> combineStyle()
```

Continue to [Expose Styleable Parts](/docs/learn/expose-styleable-parts/) to
decide which slots are public enough for consumers to import.
