---
title: Hover, Focus, And Media
description: Add local behavior without turning style objects into mini languages.
---

The same button now needs hover, focus, and mobile behavior.

You could put selectors and media rules inside one object:

```ts
{
  padding: '8px 12px',
  ':hover': { opacity: 0.86 },
  ':focus-visible': { outline: '2px solid currentColor' },
  '@media (max-width: 700px)': { width: '100%' },
}
```

That works.

The tradeoff is readability. The object started as declarations, then became a
place for state and conditions too.

Fluentic lets the object say what the element looks like, and the chain say
when it changes:

```ts
const saveButton = style({
  border: 0,
  borderRadius: 8,
  padding: '8px 12px',
}).hover({
  opacity: 0.86,
}).focusVisible({
  outline: '2px solid currentColor',
  outlineOffset: 2,
}).media('(max-width: 700px)', {
  width: '100%',
});
```

Read it as:

- object: what the element looks like;
- chain: when that changes.

Use chains for behavior that belongs to the same element.

When styles are repeated across a page, the next step is not a component API
yet. It is just moving shared styles into a file.
