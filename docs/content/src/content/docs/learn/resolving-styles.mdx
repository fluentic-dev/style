---
title: Resolving Styles
description: Resolve a component's base slots, variants, themes, and root css prop.
---

A reusable component usually has several styling inputs:

- base slot styles from the component;
- variant scopes from props;
- an incoming `theme` from outside;
- a `css` prop for root-element overrides from the caller.

Do not make each input use a different mechanism.

Resolve them together inside the component:

```tsx
const disabledButton = style.scope([
  buttonStyles.root({ opacity: 0.5, cursor: 'not-allowed' }),
]);

export function Button(props: ButtonProps) {
  const css = combineStyle(
    buttonStyles,
    props.disabled && disabledButton(buttonStyles.root),
    bindScope(buttonStyles.root, props.theme),
  );

  return (
    <button css={[css.root, props.css]} disabled={props.disabled}>
      {props.icon ? <span css={css.icon}>{props.icon}</span> : null}
      <span css={css.label}>{props.children}</span>
    </button>
  );
}
```

The important split is:

```txt
combineStyle(...) resolves slot-aware inputs.
css={[css.root, props.css]} lets caller root styles override component root styles.
```

`props.theme` can affect any public slot.

`props.css` affects only the rendered root element.

This keeps the component API understandable:

- use `theme` for component-aware customization;
- use `css` when the caller needs to override the root element;
- keep private DOM private.

Once this boundary is clear, variants, parent state, responsive changes, and
nested component themes become variations of the same pattern.
