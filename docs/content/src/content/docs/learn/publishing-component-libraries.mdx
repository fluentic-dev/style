---
title: Publishing Component Libraries
description: Treat slots, themes, and tokens as part of the public component contract.
---

A component library does not only publish behavior.

It also publishes the ways other teams are allowed to change how components
look.

That styling surface becomes part of the public contract, even when it is not
written down.

For every published component, answer the same questions:

- which parts are public slots;
- which styles remain private;
- which tokens the component reads;
- which scopes can be passed as `theme`;
- where nested component themes need to be forwarded.

When the public surface is clear, export it deliberately:

```ts
export const ButtonStyle = exposeStyle({
  root: styles.root,
  icon: styles.icon,
  label: styles.label,
});
```

Accept the right styling props:

```ts
type ButtonProps = {
  css?: StyleProp;
  theme?: StyleTheme;
};
```

Document `css` as the prop for root-element overrides.

Document `theme` as slot-aware customization.

Document tokens as shared values the component responds to.

Consumers get supported customization.

Your DOM does not become the contract.

The final guide turns those ideas into day-to-day choices.
