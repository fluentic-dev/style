title: Style One Element
description: Use style() with the css prop.
---

If Fluentic is not installed yet, start with [Installation](/docs/installation/).
This page assumes the `css` prop is already enabled.

Start with one element:

```tsx
import { style } from '@fluentic/style';

const saveButton = style({
  border: 0,
  borderRadius: 8,
  cursor: 'pointer',
  padding: '8px 12px',
});

export function SettingsPage() {
  return <button css={saveButton}>Save changes</button>;
}
```

That is enough when the style belongs to this element.

That is it: define a style, then pass it to the `css` prop.

If your team prefers class-name syntax, Fluentic supports that too. The call
arguments look different from an object style chain, but selector chains like
`.hover(...)` and `.md(...)` stay familiar, and both forms produce a style value
for the same `css` prop:

```tsx
const saveButton = style({
  display: 'inline-flex',
  alignItems: 'center',
  paddingInline: 16,
});

const saveButtonClassName = cx(
  'inline-flex',
  'items-center',
  'px-4',
);
```

Class-name chains need a transform or preset that knows what each class name
means. For Tailwind-like class names, use the
[Tailwind Class Name Preset](/docs/presets/tailwind/class-name/). For the
builder shape and custom transform details, see
[Class Name Style Chains](/docs/beyond-the-basics/class-name-style-chains/).

## Reuse Style Objects

When several styles share the same object fields, `style.plain(...)` gives you a
small reusable object that can be spread into `style(...)`.

```ts
const roundedSurface = style.plain({
  backgroundColor: '#ffffff',
  borderRadius: 8,
});

const card = style({
  ...roundedSurface,
  padding: 16,
});
```

Under the hood, `style.plain(...)` returns the same object provided to it. It is
the same as writing the reusable object directly:

```ts
const roundedSurface = {
  backgroundColor: '#ffffff',
  borderRadius: 8,
};
```

The benefit is in your editor: TypeScript checks the object against the style
type used by `style(...)`, and autocomplete can suggest valid fields.

Use `style.raw(...)` when the reusable object mixes normal CSS values with
Fluentic values such as tokens. More details on tokens come later in
[Theming With Tokens](/docs/learn/theming-with-tokens/).

```ts
import { createToken, style } from '@fluentic/style';

const surfaceColor = createToken('#ffffff');

const surface = style.raw({
  backgroundColor: surfaceColor,
});

const card = style({
  ...surface,
  padding: 16,
});
```

Continue to [Compose Element Styles](/docs/learn/compose-element-styles/) to
learn how multiple styles on one element are ordered and resolved.
