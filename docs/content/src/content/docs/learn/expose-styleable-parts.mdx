---
title: Expose Styleable Parts
description: Export only the component slots consumers may depend on.
---

You have seen how scopes can cover many component styling cases once a
component has slots.

The next decision is which slots should be available outside the component.
Some slots are part of the styling surface you want consumers to depend on.
Other styles are only implementation details.

A component can keep both kinds side by side:

```ts
const styles = {
  root: style.slot({ borderRadius: 16 }),
  title: style.slot({ fontWeight: 700 }),
  body: style.slot({ lineHeight: 1.6 }),
  measurementLayer: style({ position: 'absolute' }),
};
```

Expose only the parts consumers may rely on:

```ts
import { exposeStyle } from '@fluentic/style/utils';

export const CardStyle = exposeStyle({
  root: styles.root,
  title: styles.title,
  body: styles.body,
});
```

Outside code targets the exported surface:

```ts
export const featuredCard = style.scope([
  CardStyle.root({ backgroundColor: '#111827', color: 'white' }),
  CardStyle.title({ letterSpacing: 0.2 }),
]);
```

`exposeStyle(...)` takes an object of styles and keeps only slots and tokens.
Other values are left out. We will cover tokens in
[the next page](/docs/learn/theming-with-tokens/).

Nested fields and nested objects are supported too, so grouped slots can stay
grouped in the exposed surface.

This makes the outside styling surface explicit:

- `CardStyle` contains only the parts outside code should style;
- internal fields such as `measurementLayer` are not exposed;
- consumers target stable slot names such as `CardStyle.root`,
  `CardStyle.title`, and `CardStyle.body`;
- the component can change private styles without changing the supported styling
  surface.

Continue to [Theming With Tokens](/docs/learn/theming-with-tokens/) to see how
to make component values themeable, not only component parts styleable.
