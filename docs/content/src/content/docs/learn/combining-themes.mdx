---
title: Combining Themes
description: Keep token themes and component scopes separate, then compose them in JSX.
---

A real app often needs both kinds of theme at the same time.

A token theme changes shared values:

```ts
const darkTheme = createTheme([
  color.surface('#111827'),
  color.text('white'),
]);
```

A component scope changes public slots on one component:

```ts
const marketingCard = style.scope([
  CardStyle.root({ borderColor: '#60a5fa' }),
  CardStyle.title({ color: '#60a5fa' }),
]);
```

Do not force those into one theme object.

Compose them where they belong:

```tsx
export function App() {
  const css = combineStyle(appStyles);

  return (
    <main css={[darkTheme, css.root]}>
      <Card theme={marketingCard} title="Campaign">
        Launch plan
      </Card>
    </main>
  );
}
```

Inside `Card`, resolve the component theme against the slots `Card` owns:

```tsx
const cardCss = combineStyle(
  styles,
  bindScope(styles.root, props.theme),
);
```

This keeps each concern in the right place:

- the app root applies shared token values;
- the component receives slot-aware overrides;
- `Card` decides where those overrides attach;
- JSX puts the resolved styles on the rendered elements.

The result is flexible without turning the app theme into a giant nested object.
