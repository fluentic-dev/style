---
title: Separating Behavior From Declarations
description: Behavior needed somewhere to live without making declaration objects do everything.
---

A style object often starts simple.

Then the element needs hover, focus, media queries, supports rules, or a custom
selector.

Those are not new declarations. They describe when declarations apply.

The chain is not mainly a syntax preference.

It is a way to keep behavior out of the declaration object.

Hover, focus, media, supports, container, and custom selectors all describe
when declarations apply. They are not declarations themselves.

Moving them into chain methods keeps the object plain.

It also gives behavior a consistent TypeScript surface.

That is what later makes custom selector vocabularies possible.

The style object keeps describing style. The chain describes when that style
applies.

The next question is what happens when another team needs to style one part of
a reusable component.
