---
title: Resolving At Component Boundaries
description: Render time is where style inputs have enough context to meet.
---

A component rarely renders with one style.

It has local styles.

It may receive an app theme, a component theme, conditional styles, responsive
changes, dynamic props, and parent composition.

At some point, all of those pieces need to become one final style.

The component boundary is the first place with enough context to do that.

React renders components.

Style inputs need a place to meet too.

That leads to the next question: what identity needs to survive so the result
can still be debugged later?
