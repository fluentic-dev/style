---
title: Debugging Through Identity
description: Preserve enough identity for generated CSS to lead back to authored decisions.
---

Generated CSS is not the authoring model.

When something looks wrong, the useful question is:

```txt
which authored decision produced this rule?
```

Fluentic keeps identity on the values that matter:

- `style()` identifies an element style;
- `style.slot()` identifies a component part;
- `style.scope()` identifies a grouped override;
- tokens and values identify shared value sources;
- JSX element markers identify where `css` attached.

Sourcemap modes exist because spread and merge sources can be read in two useful
ways:

```txt
style mode -> point to the leaf style call that assembled the rule
value mode -> point to the original reusable value when possible
```

Debugging works because identity is part of the data model, not an afterthought
added after CSS strings are generated.
