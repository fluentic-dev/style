---
title: Runtime And React
description: Preserve React composition when style decisions are still dynamic.
---

Static extraction can move a lot of style work out of the browser.

React applications are still dynamic.

A component may receive props, themes, conditions, or library code that the
build never saw in its final app context.

Some styling decisions still happen at render time:

- values known only at render time;
- conditional scopes;
- component libraries;
- app-specific theme composition;
- runtime-only development and tests.

Runtime is normal React work.

It is the part that handles styling decisions the build cannot know because the
final component tree only exists while the app renders.

That raises the build question from the other side: which work can move out of
the browser before the app runs?
