---
title: Publishing Less Surface Area
description: Keep public styling APIs smaller than a component's internal style object.
---

A component often has more internal styles than public styling promises.

It may have a root and title that callers can style.

It may also have wrappers, measurement layers, layout helpers, or temporary
elements that should stay private.

Public slots are part of the contract.

Private styles are implementation.

A narrowed export makes that boundary visible.

Consumers see the supported styling surface, not every internal style value.

Use `exposeStyle()` for that export.

```ts
import { style } from '@fluentic/style';
import { exposeStyle } from '@fluentic/style/utils';

const styles = {
  root: style.slot({
    borderRadius: 12,
  }),
  title: style.slot({
    fontWeight: 700,
  }),
  body: style.slot({
    lineHeight: 1.6,
  }),
  measurementLayer: style({
    position: 'absolute',
    visibility: 'hidden',
  }),
  layoutHelper: style({
    display: 'grid',
  }),
};

export const CardStyle = exposeStyle(styles);
```

The component can still use the full internal `styles` object.

Consumers import `CardStyle`.

```ts
import { style } from '@fluentic/style';
import { CardStyle } from './card';

export const featuredCard = style.scope([
  CardStyle.root({
    borderColor: '#2563eb',
  }),
  CardStyle.title({
    color: '#2563eb',
  }),
]);
```

`CardStyle` exposes `root`, `title`, and `body`.

It does not expose `measurementLayer` or `layoutHelper`, because those are plain
`style()` values, not slots.

`exposeStyle()` also works with nested style objects and tokens.

```ts
const styles = {
  tokens: cardTokens,
  root: style.slot({
    backgroundColor: cardTokens.surface,
  }),
  text: {
    title: style.slot({
      color: cardTokens.title,
    }),
    helper: style({
      opacity: 0.72,
    }),
  },
};

export const CardStyle = exposeStyle(styles);
```

The exposed object keeps `tokens`, `root`, and `text.title`.

It drops `text.helper`.

That is the point: public slots and tokens are the styling contract; private
styles remain an implementation detail.

`exposeStyle()` is not a security boundary.

It is an API boundary for TypeScript, docs, and component-library consumers.

This mirrors normal component design: export what callers may depend on, then
keep the rest free to change.
