---
title: Plugin Options
description: Configure compiler behavior and transform filters.
---

Bundler plugins accept compiler options and transform filter options.

```ts
stylePlugin({
  hoist: true,
  include: /\.[jt]sx?$/,
  exclude: /node_modules/,
  getSourcemapFilePath({ relativePath }) {
    return `/src/${relativePath}`;
  },
});
```

## `hoist`

Controls whether extracted style, slot, and scope builders are hoisted to module
scope. Dynamic inline values are carried through generated token bindings.

```ts
hoist?: boolean
```

The default is `true`. Set `hoist: false` to keep extracted helper creation at
the original call site.

## `include` and `exclude`

Control which files are transformed.

```ts
include?: RegExp | string | string[]
exclude?: RegExp | string | string[]
```

The default include is JavaScript and TypeScript files. The default exclude is
`node_modules`.

## `getSourcemapFilePath`

Customize the source path or URL written into CSS sourcemaps for a transformed
source file.

```ts
getSourcemapFilePath?: (info: {
  filePath: string;
  absolutePath: string;
  relativePath: string;
  rootDir?: string;
  sourceUrl?: string | null;
}) => string | null | undefined;
```

Use this when a framework serves browser sourcemap sources under a different
path than the absolute file path seen by the plugin.

```ts
stylePlugin({
  getSourcemapFilePath({ relativePath }) {
    return `webpack://_N_E/./${relativePath}`;
  },
});
```

Return `null` or `undefined` to keep the integration default.

## `importSources`

Register style functions that the compiler should extract.

```ts
importSources?: {
  source?: string | RegExp | (string | RegExp)[];
  name?: string | string[];
  styleFn: StyleFn;
  check?: (info: { source: string; name: string }) => boolean;
}[]
```

`source` accepts exact import sources, glob patterns, regular expressions, or an
array of patterns. Use this when an app has multiple style functions:

```ts
stylePlugin({
  importSources: [
    { source: './style', name: 'style', styleFn: style },
    { source: './sx-*', name: 'sx', styleFn: sx },
    { source: /ui-style$/, name: 'ui', styleFn: ui },
  ],
});
```

Each imported builder keeps its own selectors and transform during extraction.

## `css`

Configure generated CSS behavior.

```ts
css?: {
  layer?: boolean;
}
```

Additional generated CSS config can be passed through this object.
