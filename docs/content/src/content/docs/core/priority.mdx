---
title: Priority
description: Understand how value weights, selectors, scopes, media rules, and properties are ordered.
---

Fluentic emits atomic CSS in a stable priority order. The order is nested, not a
single flat score.

Lower buckets are emitted first. Higher buckets win later in the cascade.

```txt
value weight
  direct selector bucket
    parent selector bucket
      media bucket
        property bucket
```

## Value Weight

Explicit value weight is the outermost override layer.

```ts
const strong = style({
  color: [1, 'red'],
});
```

A higher value weight wins over any lower value weight, even if the lower value
is inside a selector or media rule.

If two declarations have the same value weight, Fluentic falls through to the
remaining buckets. Value weight is the escape hatch; equal weights still keep
the normal selector, media, and property rules.

```txt
value weight 1 base
value weight 1 base media
value weight 1 state selector
value weight 1 state selector media
value weight 2 base
```

## Selector Buckets

Selector buckets keep state rules from being crossed by unrelated media rules.

```txt
base
parent selector
direct selector
parent + direct selector
priority direct selector
```

Parent selectors come from scopes. Direct selectors come from state and
self-selector methods like `.hover(...)`, `.active(...)`, `.focus(...)`, and
`.select(...)`.

```ts
const card = {
  button: style.slot({ color: 'black' }).active({ color: 'red' }),
};

const parentHover = style.scope().hover([
  card.button({ color: 'blue' }),
]);
```

The parent hover rule is contextual. It cannot beat the direct button active
bucket. If you want to override direct active inside the parent state, write the
same direct state inside the scope:

```ts
const parentHoverButtonActive = style.scope().hover([
  card.button().active({ color: 'purple' }),
]);
```

That produces a parent + direct selector bucket, which is narrower than the
direct selector alone.

The order of priority selectors is configurable. The default preset orders
common states like this:

```txt
link -> visited -> hover -> focusWithin -> focus -> focusVisible -> active -> disabled
```

That means `active` wins over `hover` in the default `style` export. Design
systems can change this order with `selectorPriority(...)` when creating a
custom style function.

## Media Buckets

Media is a convenience override inside the current selector bucket.

```txt
base
base media
base weighted media

state selector
state selector media
state selector weighted media
```

This means base media can override base, but base media cannot override a state
selector like hover, active, focus, or a custom selected state.

```ts
const button = style({
  color: 'black',
}).media('(max-width: 700px)', {
  color: 'blue',
}).hover({
  color: 'red',
});
```

At small widths, the media rule can override the base color. It still does not
override the hover color.

To override a state in media, write the media rule in the same state bucket:

```ts
const button = style({
  color: 'black',
}).hover({
  color: 'red',
}).media(
  '(max-width: 700px)',
  style({
    color: 'blue',
  }).hover({
    color: 'purple',
  }),
);
```

Now the media block has a base media override and a hover media override. The
base media color can override the base color, and the hover media color can
override the base hover color at the matching width. The same idea applies to
`active`, `focusVisible`, custom state selectors, and scoped parent + direct
selector combinations.

Weighted media only wins inside the same selector bucket:

```ts
const panel = style().media(2, '(max-width: 700px)', {
  padding: 12,
});
```

Weighted base media still cannot cross into a direct selector bucket.

## Property Bucket

Property priority is the innermost bucket:

```txt
shorthand
intermediate
longhand
```

This keeps longhand declarations after the shorthand declarations they refine.

```ts
const box = style({
  margin: 4,
  marginTop: 8,
});
```

`marginTop` is emitted after `margin`.

## Full Order

The practical order looks like this:

```txt
value weight 0
  base
    base media
    base weighted media

  parent selector
    parent selector media
    parent selector weighted media

  direct selector
    direct selector media
    direct selector weighted media

  parent + direct selector
    parent + direct selector media
    parent + direct selector weighted media

  priority direct selector
    priority direct selector media
    priority direct selector weighted media

value weight 1
  ...same expansion...

value weight 2
  ...same expansion...
```

Use value weight as the explicit escape hatch. Use media for responsive
overrides inside the same base, parent, or direct selector context.
