Use the SSR style benchmark when you want a render-only server benchmark for
runtime CSS-in-JS style resolution work.

```sh
pnpm bench:ssr-style
```

This benchmark is inspired by Goober-style render benchmarks. It compares
Goober, styled-components, Emotion, Fluentic dynamic runtime style creation, and
Fluentic hoisted runtime style switching.

## What It Is

The report measures `renderToString()` style resolution work for runtime styling
patterns.

It is useful for asking:

| Question | Why this report helps |
| --- | --- |
| How expensive is runtime style resolution during server render? | The benchmark keeps the render loop focused on style resolution. |
| Does hoisting styles change the server render cost? | Fluentic has both dynamic and hoisted lanes. |
| How do runtime libraries behave in a similar render loop? | The compared libraries use matching scenario data. |

## What It Is Not

This is not a full SSR CSS extraction benchmark. It does not measure server
stylesheet collection APIs, static CSS module output, extracted CSS delivery, or
browser hydration.

Keep these numbers separate from the React app benchmark and the bundle-size
report.
