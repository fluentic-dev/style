Use the React app benchmark when you want an app-shaped timing report instead of
a tiny style-only loop.

```sh
pnpm bench
```

For more reportable local numbers, use the stable run:

```sh
pnpm bench:stable
```

For larger row-count scenarios, use the stress run:

```sh
pnpm bench:stress
```

## What It Measures

The benchmark runs the same dashboard/table shape across styling approaches. It
keeps the DOM shape, data, viewport, keys, React version, and browser automation
aligned so the report is about the styling path as much as possible.

The default Fluentic rows use extracted production output and public authoring
patterns:

- direct css prop: stable `style.slot()` values declared outside render and
  passed directly as `css={styles.item}`;
- scoped composition: stable slots and scopes declared outside render, composed
  with `combineStyle()` and `bindScope()`;
- token composition: the same scoped composition plus a bounded set of token
  overrides for theme-like values.

Open-ended per-row values are passed through the normal React `style` prop for
every app. Fluentic no-hoist, inline dynamic style creation, and runtime-only
css-prop variants are diagnostic lanes, not part of the default comparison.

The report includes:

| Metric | Meaning |
| --- | --- |
| `initialMountMs` | First render for a scenario. |
| `updateMs` | Known prop/state updates against the existing tree. |
| `remountMs` | Recreating the dashboard tree. |
| `routeSwitchMs` | Switching between dashboard views. |
| `styleTags` | Number of style tags after the run. |
| `cssRules` | Number of stylesheet rules after the run. |

Median is the headline timing. Mean, p95, cold first-run mount, and raw samples
stay in the JSON report so regressions can be inspected later.

## Lanes

Fluentic is intentionally reported in separate modes:

| Lane | Meaning |
| --- | --- |
| `fluentic-style-extract-direct` | Extracted production css-prop path with stable slots passed directly. |
| `fluentic-style-extract-scoped` | Extracted production path with stable scopes composed through `combineStyle()` and `bindScope()`. |
| `fluentic-style-extract-token` | Extracted production scoped path plus bounded token override values. |
| Fluentic diagnostic lanes | No-hoist, inline dynamic style creation, and runtime-only css-prop variants. They are excluded by default; select them explicitly or set `INCLUDE_INTERNAL=1`. |

Other apps include Emotion, styled-components, Goober, StyleX, vanilla-extract,
CSS Modules, and an experimental Panda CSS app.

## Filtering

```sh
APP=fluentic-style-extract-direct,emotion pnpm bench
ROWS=100,500,1500 pnpm bench
REPEATS=5 WARMUPS=3 MEASURED=30 pnpm bench
INCLUDE_INTERNAL=1 pnpm bench
INCLUDE_EXPERIMENTAL=1 pnpm bench
```

Use filtering when you are checking one regression or one competitor lane. Use
the full run when publishing a local comparison.
