Use the React app benchmark when you want an app-shaped browser timing report
instead of a tiny style-only loop.

```sh
pnpm bench
```

For more reportable local numbers, use the stable run:

```sh
pnpm bench:stable
```

For larger row-count scenarios, use the stress run:

```sh
pnpm bench:stress
```

## What It Measures

The benchmark runs the same dashboard/table shape across styling approaches. It
keeps the DOM shape, data, viewport, keys, React version, and browser automation
aligned so the report is about the styling path as much as possible.

The default Fluentic rows use extracted production output and public authoring
patterns:

- direct DOM: stable `style.slot()` values declared outside render and passed
  directly as `css={styles.row}`;
- scoped DOM: stable slots, scopes, and token-backed themes declared outside
  render, composed with `combineStyle()` and `bindScope()`;
- direct components: row/cell work split into React components, each using its
  own stable local styles;
- scoped components: componentized rows using stable scopes and theme tokens.

The main suite does not use ever-changing per-row token overrides. Theme/token
links are still present through stable scopes and status themes. Open-ended
high-cardinality values belong in separate diagnostics because browsers spend
extra work invalidating CSS custom properties in the real DOM.

Fluentic benefits from stable module-scope style objects and cache reuse across
repeated component instances. That is intentional and mirrors long-lived browser
apps. For the underlying resolver model, see
[Style Resolution And Cache](../design/style-resolution-and-cache/).

The report includes:

| Metric | Meaning |
| --- | --- |
| `initialMount` | First render for a scenario. |
| `updateStyle` | Known visual variant updates against the existing tree. |
| `remount` | Recreating the dashboard tree. |
| `route` | Switching between dashboard views. |
| `styleTagCount` | Number of runtime style tags after the run. |
| `ruleCount` | Number of stylesheet rules after the run. |

Median is the headline timing. Mean, p95, cold first-run mount, and raw samples
stay in the JSON report so regressions can be inspected later.

## Filtering

```sh
APP=fluentic-style-extract-direct-dom,stylex pnpm bench
ROWS=100,500,1500 pnpm bench
REPEATS=5 WARMUPS=3 MEASURED=30 pnpm bench
INCLUDE_EXPERIMENTAL=1 pnpm bench
```

Use filtering when you are checking one regression or one competitor lane. Use
the full run when publishing a local comparison.
