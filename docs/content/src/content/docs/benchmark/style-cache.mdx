title: Style Cache

---

Use the style-cache benchmark when you want to inspect the warmed runtime paths
that happen after an app has already created common styles.

```sh
pnpm bench:fluentic-cache
```

The benchmark runs in the browser and rotates variant order across repeats. Each
cross-library variant runs in a fresh browser context so one library family does
not warm another library's state.

## What It Covers

| Variant family | What it exercises |
| --- | --- |
| Fluentic `combineStyle()` cache | Parent/child cache shapes and repeated composed styles. |
| Fluentic css-prop cache | Reusing the same css-prop inputs across many elements. |
| Fluentic no-css JSX path | Custom JSX runtime overhead when no css prop is present. |
| Precomputed class names | Rendering with already-resolved className output. |
| Runtime library variants | Hoisted and inline dynamic Goober, Emotion, and styled-components cases. |

```sh

ITEMS=5000 BENCH_OUT_DIR=./tmp/cache-report pnpm bench:fluentic-cache
```

Use this report for runtime cache behavior only. Do not mix it with the React app
dashboard numbers; they measure different bottlenecks.
