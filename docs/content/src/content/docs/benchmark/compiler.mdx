Use the compiler benchmark when you want to measure Fluentic's transform cost
across generated source shapes.

```sh
pnpm bench:compiler
```

The report writes JSON under `benchmark/main/results` by default, or under
`BENCH_OUT_DIR` when that variable is set.

## What It Measures

The compiler benchmark generates source corpora, runs the Fluentic transform,
and records transform timing. It is useful when changing extraction, evaluator,
or sourcemap behavior.

Use it for questions like:

| Question | Example change |
| --- | --- |
| Did extraction get slower for many files? | Evaluator or import-resolution changes. |
| Did a source pattern become expensive? | New chain methods, spreads, merges, or tokens. |
| Does sourcemap/debug work change transform time? | Debug transform or source-location changes. |

## Settings

```sh
FILES=200 COMPONENTS=20 pnpm bench:compiler
WARMUPS=3 MEASURED=20 pnpm bench:compiler
VARIANTS=theme-slots:fluenticExtractWarm pnpm bench:compiler
BENCH_OUT_DIR=./tmp/compiler-report pnpm bench:compiler
```

Run this benchmark before and after compiler-heavy changes on the same machine.
The before-and-after delta matters more than a single absolute number.
